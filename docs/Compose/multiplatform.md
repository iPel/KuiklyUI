## 多端适配与平台差异

本页目标：从 **Android / iOS / HarmonyOS / Web / 小程序** 五端视角，总结 Kuikly Compose 在行为上的差异与注意事项。

> 各端的接入与调试步骤，已在 Kuikly QuickStart 中有详细说明，这里只从 Compose 视角补充说明。

### 支持状态概览

- **Android**：Compose 的原生阵地，能力最完整
- **iOS**：通过 Kuikly Core + iOS Render 模块映射到 UIKit
- **HarmonyOS**：通过 Kuikly Core + Harmony Render 映射到 ArkUI
- **Web**：映射到 DOM / WXML 结构
- **小程序**：映射到小程序原生组件

整体目标：**保证主要布局与交互行为在各端尽可能一致**，在个别平台受限的能力上给出明确说明与降级策略。

### 需要特别关注的差异类型

1. **字体与排版**
   - 各端默认字体与排版引擎不同，极端场景下可能出现换行位置和 ellipsize 表现差异
   - 建议在视觉上为边界情况（极长文本、Emoji 混排）预留空间

2. **滚动物理与回弹**
   - iOS / HarmonyOS 通常有更明显的回弹效果
   - Web / 小程序环境会受宿主容器限制
   - Kuikly 在 List / Scroll 相关组件中提供了 bounces 等能力开关，不同端的具体表现会在「列表与滚动专题」中详列

3. **输入法与键盘**
   - 不同平台的键盘弹出/收起时对布局影响不完全一致
   - 建议对重要表单页面进行多端真机验证，必要时针对平台做局部适配

4. **系统 UI 与安全区域**
   - 顶部状态栏、底部操作栏在不同平台的高度与表现不同
   - 使用 Scaffold / 自定义容器时，建议显式处理安全区域（后续会提供具体示例）

### 调试与排查建议

- Android：使用 Android Studio 与 Profiler 调试 Compose 层性能与重组情况
- iOS：使用 Xcode / Instruments 观察渲染与内存情况
- Web：使用浏览器 DevTools 分析布局与性能
- 小程序：使用小程序开发工具进行调试与性能分析

具体到 Kuikly 层的调试能力（日志、性能分析工具等），请参考：

- `/DevGuide` 下相关调试文档
- KuiklyBase 相关性能与可观测性文档
